# Инструкция по обновлению сайта (Deployment Guide)

Этот файл содержит шаги для ручного обновления сайта на хостинге после внесения изменений в репозиторий GitHub.

## Предварительные требования
1.  У вас настроен SSH-доступ к серверу.
2.  Вы находитесь в консоли сервера.

## Настройка подключения (Если репозиторий еще не подключен)

Если вы делаете это **в первый раз** и папка с сайтом уже существует на хостинге:

1.  Перейдите в папку сайта:
    ```bash
    cd ~/www/site_1
    ```
2.  Инициализируйте Git:
    ```bash
    git init
    ```
3.  Подключите удаленный репозиторий:
    ```bash
    git remote add origin https://github.com/majormguarde-bit/tzavru.git
    ```
4.  Скачайте код (принудительно, если файлы уже есть):
    ```bash
    git fetch origin
    git reset --hard origin/master
    ```

### Авторизация (Важно!)
При выполнении команд Git попросит логин и пароль.
*   **Username**: `majormguarde-bit`
*   **Password**: Здесь нужно использовать **Personal Access Token**, а не пароль от аккаунта (так как GitHub отключил поддержку паролей).

**Альтернатива (SSH):**
Если у вас настроены SSH-ключи на сервере, лучше использовать SSH-ссылку:
```bash
git remote set-url origin git@github.com:majormguarde-bit/tzavru.git
```

## Пошаговая инструкция (Регулярное обновление)

### 1. Переход в папку проекта
Перейдите в директорию, где размещен ваш сайт. Путь может отличаться в зависимости от настроек хостинга.

```bash
# Пример для cPanel / обычного хостинга
cd ~/www/site_1
# ИЛИ
cd ~/public_html
```

### 2. Получение обновлений (Git Pull)
Загрузите последние изменения из репозитория GitHub.

```bash
git pull origin master
```

> **Примечание:** Если возникают ошибки из-за локальных изменений в конфигурационных файлах, вы можете временно скрыть их:
> ```bash
> git stash
> git pull origin master
> git stash pop
> ```

### 3. Обновление зависимостей (Python Packages)
Если файл `requirements.txt` был изменен, необходимо обновить библиотеки в виртуальном окружении.

```bash
# Активация виртуального окружения (путь может быть venv, env, .venv)
source venv/bin/activate

# Установка/обновление пакетов
pip install -r requirements.txt
```

### 4. Применение изменений базы данных (Миграции)
Проект использует **Flask-Migrate** (Alembic) для управления схемой базы данных.

**ВАЖНО:** После обновления кода необходимо применить миграции, чтобы обновить структуру таблиц (добавить новые колонки и т.д.).

```bash
# Убедитесь, что виртуальное окружение активировано
# source venv/bin/activate

# Применение миграций
flask db upgrade
```

> Если команда `flask` не найдена, попробуйте `python -m flask db upgrade`.

### 5. Перезапуск приложения
Чтобы изменения вступили в силу, перезапустите веб-сервер.

**Для Passenger (cPanel, ISPmanager):**
```bash
mkdir -p tmp
touch tmp/restart.txt
```

**Для Systemd (VPS, Gunicorn):**
```bash
sudo systemctl restart site_1
```

## ⚠️ ПОЛНАЯ ПЕРЕУСТАНОВКА (Сброс и очистка базы)

Если возникли критические ошибки с базой данных (например, рассинхронизация миграций) и вы хотите начать **с чистого листа** (ВНИМАНИЕ: Все данные будут удалены!):

1. **Перейдите в папку проекта:**
   ```bash
   cd ~/tzav/default
   ```

2. **Сбросьте код до состояния репозитория (удалит локальные изменения):**
   ```bash
   git fetch origin
   git reset --hard origin/master
   ```

3. **Удалите старую базу данных и временные файлы:**
   ```bash
   rm -rf instance/*.db
   rm -rf *.db
   find . -name "*.pyc" -delete
   find . -name "__pycache__" -delete
   ```

4. **Обновите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

5. **Создайте новую базу данных:**
   ```bash
   python init_db.py
   ```

6. **Создайте администратора (Login: admin / Pass: admin123):**
   ```bash
   python create_admin.py
   ```

7. **Синхронизируйте миграции (чтобы в будущем работали):**
   ```bash
   flask db stamp head
   ```

8. **Перезапустите сайт:**
   ```bash
   touch tmp/restart.txt
   ```
